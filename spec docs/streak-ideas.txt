Good instinct—but don’t go pure streak-based leveling without guardrails. It feels amazing when it works, but it can also punish players hard on a single mistake.

The sweet spot is:

streak drives intensity + rewards, while progression remains stable

Here’s a tight design you can actually ship.

🎯 Core Model

Separate these three:

1. Score (long-term progression)
Persistent within a run
Determines level unlocks
2. Streak (short-term performance)
Resets on failure
Drives multipliers + tempo
3. Flow State (new layer)
Derived from streak
Controls difficulty in real time
🔥 Option A — Pure Streak Levels (what you’re considering)
Model
level = floor(streak / N)

Example:

0–4 → Level 1
5–9 → Level 2
10–14 → Level 3
Pros
Immediate feedback loop
Feels skill-based
High tension
Cons (serious)
One mistake = total collapse
Players play safe, not musical
Hard to balance long sessions
⚖️ Recommended Hybrid Model
1. Base Level (Score-driven)
baseLevel = f(score)
Never decreases
Controls:
unlocks
max system complexity
available modules
2. Streak Tier (Performance layer)
streakTier = floor(streak / tierSize)

Example:

Tier 0: 0–2
Tier 1: 3–5
Tier 2: 6–9
Tier 3: 10+
3. Effective Difficulty
effectiveLevel = baseLevel + streakTier

This is what actually drives:

chord difficulty
timing pressure
reward scaling
🎮 What This Feels Like
You climb slowly (score)
You spike hard when playing well (streak)
You don’t get punished permanently for a miss

This creates:

“I’m in the zone → things are getting intense → I drop → I recover quickly”

💰 Reward Structure (this is the fun part)
Base Formula
reward =
  baseScore
  * difficultyMultiplier
  * streakMultiplier
  + timeBonus
1. Streak Multiplier (core driver)
streakMultiplier = 1 + (streak * k)

Clamp it:

streakMultiplier <= maxMultiplier

Example feel:

streak 0 → x1.0
streak 5 → x1.75
streak 10 → x2.5
2. Tier Bonus (discrete spikes)

Each tier adds a kicker:

tierBonus = streakTier * tierValue

This makes crossing thresholds feel important

3. Difficulty Multiplier
difficultyMultiplier = 1 + (chordDifficulty * d)

Encourages:

not just speed
but complexity
4. Time Pressure Bonus

Keep it simple:

timeBonus = remainingTimeRatio * maxBonus
🧠 Key Design Trick (this matters a lot)
Soft Streak Decay (instead of hard reset)

Instead of:

streak = 0

Try:

streak = floor(streak * 0.5)

or:

streak -= 2
Why this is huge
Keeps momentum alive
Reduces frustration
Encourages risk-taking
⚡ Flow-Based Difficulty Scaling

Tie gameplay pressure to streak:

As streak increases:
hint duration ↓
time limit ↓
chord complexity ↑
visual intensity ↑

Example:

effectiveHintTime =
  baseHintTime * (1 - streakTier * 0.1)
🧩 How This Hooks Into Your Systems
Patch System
Higher streak tiers unlock:
more modules temporarily
extra modulation routes
Lose streak → system simplifies
Shop / Unlocks
Score unlocks permanent tools
Streak unlocks temporary power
🎯 Concrete Example
Player state:
score → Level 3
streak → 8 → Tier 2
effectiveLevel = 3 + 2 = 5

Game reacts:

harder chords appear
rewards spike
timer tightens

Player fails:

streak → 4 (not 0)
tier → 1
effectiveLevel → 4

They’re still “in it”

🚫 What to avoid
❌ Pure streak progression (too punishing)
❌ Hidden scaling (player must feel it)
❌ Flat rewards (kills engagement)
❌ Instant resets (kills flow)
🧭 Bottom Line

Use streak to:

amplify, not define progression

Let:

score = stability
streak = intensity
effective level = gameplay feel
